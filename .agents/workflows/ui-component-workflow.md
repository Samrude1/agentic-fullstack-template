# UI Component Workflow

1. **Architect** (`/architect`):
   - Plan the component's API and look according to `ui-registry.md`.
2. **Develop**:
   - Build the component using Tailwind CSS and the defined design tokens.
3. **Review** (`/review`):
   - Verify visually and technically against design standards.
4. **Imprint** (`/imprint`):
   - Add the new component patterns to `ui-registry.md` to maintain consistency across the app.
